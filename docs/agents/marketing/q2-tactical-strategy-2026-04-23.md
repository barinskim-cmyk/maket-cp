## 1. TL;DR

Q2 для GS — это не маркетинговый запуск, а **подготовка photographer-led motion к Q3** на минимально достаточных материалах. Цель квартала: к концу июня иметь 30–50 фотографов в активной открытой бете, 1+ фотографа-евангелиста, который привёл бренд, 2–3 отправленные outbound-волны с замеренной конверсией и готовый [test user A] case study. Позиционирование locked на Version A; ни один внешний текст не уходит с устаревшими формулировками («бизнес-аналитик», «DAM», «EU серверы», «PIM в разработке»). Hero-badge на лендинге — гибрид: открытая бета для фотографов + закрытая «ищем 3 production-команды» для брендов; одна форма с routing по первому вопросу, email + 2–3 динамических follow-up. GTM motion — photographer-led, bottom-up; хук №1 — автопереименование. Фикстура для Chrome-agent демо — Машин **собственный** проект, не клиентский. GS не делает brand guide на 50 страниц — делает минимум для outbound + лендинга и встаёт на release review только когда в скоупе копирайт. Первые две недели — пара «фотограф-евангелист + бренд» (гипотезы), outbound-плейбук, поддержка DAD в landing cleanup. Риски топ-3: каналы не активируются, первый евангелист не находится, продуктовые хуки не стабилизируются вовремя.

## 2. Goal alignment

### 2.1. Ставка 5 — photographer-led motion (главный вклад GS)

Ставка 5 требует: (а) найти и подключить первого фотографа-евангелиста, (б) через него вывести первый бренд к разговору, (в) замерить «до/после» на этой паре, (г) подготовить плейбук для масштабирования в Q3. GS закрывает это в Q2 так:

- **Плейбук outbound** — каркас сообщений, хуки, CTA, tone-of-voice (Маша подписывает лично, «от-коллеги-коллеге»). Без плейбука каждая рассылка — ручная работа Маши; с плейбуком — повторяемый канал.
- **Канал-гипотезы 1+2** — личная сеть Маши и Telegram-каналы ретушёров тестируются первыми, бесплатно, с быстрым feedback. Тест по первой волне даёт ответ: «работает ли хук автопереименования в принципе».
- **Механика upward-sell** (A) share-link authority + (B) персональное интро — пассивная + активная seed. GS готовит копирайт для brand-signature в share-link (PA делает реализацию) и скрипт интро (CX проводит разговор).
- **Content seed** — видео-демо автопереименования (30–60 сек) — универсальный artefact для cold email, Telegram-поста, insertion в YouTube-канал.

### 2.2. Ставка 3 — anchor-клиенты (частичный вклад GS)

Ставка 3 ([anchor client], Беларусь-пилот, [test user A] retain + upsell) — основной owner это CX + Маша. GS закрывает **материалы**: [test user A] case study, one-pager с ценностью для anchor-планов (без цен до утверждения monetization-matrix), замер «до/после» framework, follow-up-сообщения после первого касания CX.

### 2.3. Что GS **не** делает в Q2 (явно)

Pitch-deck (это Q3 после case study готов), whitepaper, английский контент (до Беларусь-подписания), прямые холодные письма в бренды без warm-intro, ценовые офферы до monetization-matrix, brand guide полноценный (минимум для outbound+landing, не 50 страниц).

## 4. Конкретные действия GS

### 4.1. Outbound-плейбук для фотографов (skeleton + 2 шаблона)

**Цель:** превратить холодный контакт фотографа в «попробовал автопереименование на своей съёмке».
**Аудитория:** коммерческий фотограф, снимающий каталоги (e-commerce, marketplace, fashion).
**Хук №1:** автопереименование («одна кнопка — 248 IMG стали ALX-T201»).
**Хук №2 (резерв):** share-link клиенту («клиент согласовывает каталог с телефона, не запрашивает перепересылку»).
**CTA:** бесплатный доступ на одну съёмку + 15-мин демо от Маши.
**Tone:** от-коллеги-коллеге, без маркетингового глянца, на «ты», русский.
**Подпись:** Маша лично (не Maket CP, не команда) — это GTM-рычаг доверия.
**Что НЕ упоминаем:** цены, «платформа для продакшена», юр-детали, «EU серверы», «PIM», роадмап.
**QA-gate:** перед отправкой волны — QA smoke-тест share-link + автопереименования.
**Release review:** GS обязателен на любых копирайт-правках плейбука.

**Шаблон 1 — Telegram DM / личная сеть (теплый канал).**

> Привет, [имя]. Видел твой каталог [бренд / жанр] — помнишь, как там с переименованием? У меня была та же боль: 800 IMG_4201 и 2 часа ручной работы по Excel-чек-листу. Мы сделали штуку, которая делает это одной кнопкой: кидаешь PDF чек-листа, AI матчит артикулы с кадрами, все файлы переименовываются. Если интересно — пришлю 30-сек видео и подключу бесплатно на твою ближайшую съёмку. Скажи «да» — вышлю ссылку.
> — Маша

**Шаблон 2 — Telegram-каналы / открытые чаты ретушёров (холодный канал).**

> Коллеги, всем привет. Мы фотографы сами, и нам надоело переименовывать 500 файлов в конце съёмки вручную. Сделали инструмент: PDF чек-листа от бренда + 800 кадров → один клик, все имена по формату. В бете сейчас открыто, бесплатно на первую съёмку — если хотите попробовать, напишите в ЛС или в комментариях.
> 30-секундное видео-демо: [ссылка]
> Кто уже пробовал — поделитесь впечатлениями.

**Шаблон follow-up (если нет ответа через 4–5 дней):**

> [Имя], не хочу быть навязчивой — коротко: я запилила этот инструмент для себя же, сейчас отдаю бесплатно в бета-доступ тем, кто готов попробовать. Если не до этого — игнорь, без обид. Если интересно — вот видео-демо: [ссылка]. Вопросы — отвечу.

### 4.2. Канал-гипотезы (3–5 в приоритете)

- Cost-per-activated-signup < 1000 ₽ (платные каналы).

### 4.3. Механика upward-sell (требует хуков PA)

**Вариант A — Share-link authority (пассивная seed).**
Владелец открывает share-link клиента, видит в футере короткую подпись «Проект ведётся в Maket CP — платформа для визуального продакшена. [Узнать больше]». Клик уводит на landing-секцию с коротким 30-сек видео «что такое Maket CP для владельца» + форма демо-запроса.
*Что нужно от PA:* добавить brand-signature block в share-link (конфигурируемый ON/OFF по проекту), передать параметр source=share-link на landing.
*Что нужно от DAD:* дизайн signature (ненавязчивый, не перетягивает внимание с контента клиента).
*Что нужно от GS:* копирайт signature, landing-секция для входящих из share-link, 30-сек видео-tour share-link с точки зрения владельца.

**Вариант B — Персональное интро (активный ход).**
Когда фотограф-евангелист видит, что владелец открывал share-link несколько раз / задаёт вопросы / упоминает Maket в переписке — фотограф передаёт контакт владельца в CX. CX пишет follow-up от Маши в течение 24 часов: «[Фотограф] сказал, что вам было удобно на share-link — могу за 15 минут показать, что ещё видно из admin-дашборда».
*Что нужно от CX:* скрипт warm-intro follow-up, Calendly-слоты Маши.
*Что нужно от GS:* копирайт follow-up-сообщения владельцу (шаблон в разделе 4.4).
*Что нужно от продукта:* чтобы это не ломалось — share-link стабилен, Rate Setter не падает, автопереименование работает.

**Вариант C (отложен в Q4):** feature-seeded upsell в owner-dashboard — CTA «расширить до полного плана». Требует зрелого dashboard и утверждённой monetization-matrix.

## 6. Release review gate — роль GS

**Когда GS вызывается на релиз:**

| Тип релиза | GS обязателен | GS опционален | Не вызывается |
|---|---|---|---|
| Landing — копирайт или hero-message | Да | — | — |
| Landing — только визуал/CSS без правки текста | — | Да (спросить «затронут ли копирайт») | — |
| UI-строки продукта (tooltips, error messages, onboarding) | Да, если >5 новых строк или затрагивают Version A | Да, если 1–5 строк | Да, если баг-фикс без новых строк |
| Outbound-волна (любая) | **Да, всегда** | — | — |
| Blog post / social-post / external case study | **Да, всегда** | — | — |
| One-pager, pitch-deck, sales-материалы | **Да, всегда** | — | — |
| Email transactional (confirm, reset, notifications) | Да, при первом релизе или редизайне | Да, при мелких правках | — |
| Внутренние доки (CLAUDE.md, status.md, агенты-лейны) | — | — | **Нет** |
| Чистый бэкенд-релиз без user-facing копирайта | — | — | **Нет** |

**Что GS ревьюит на release gate (чек-лист):**

1. **Соответствие Version A** — core message «отражает реальный процесс команды, вы видите его и управляете по-настоящему, экономит время и деньги». Не «DAM», не «бизнес-аналитик», не «CRM для фотографов».
2. **Нет устаревших формулировок** — «EU серверы», «PIM в разработке», «бизнес-аналитик», любые claim'ы про функции, которых нет в продакшн.
3. **Tone consistency** — для фотографов «от-коллеги», для владельцев деловой с цифрами, нет глянца, нет «game-changer / revolutionary / best-in-class».
4. **CTA** — один на материал, явный, измеримый.
5. **Aud.-fit** — одно сообщение одной аудитории (фотограф ИЛИ владелец; не обоим сразу, кроме hero-badge).
6. **Reality check vs `/dev/status.md`** — не обещаем функций, которые падают в QA.

**SLA ревью:** 24 часа для outbound и landing, 48 часов для более объёмных материалов. Если GS не успевает — блокирует релиз до ревью, не молча пропускает.

## 8. Risk register

### R-GS-1: Каналы не активируются к середине мая
**Вероятность:** средняя. **Impact:** средний. Без рабочего канала — нет воронки, нет фотографов, нет pipeline для Q3.
**Митигация:** (а) тестируем 2 канала параллельно (не последовательно), чтобы не класть всё на один; (б) критерии успеха — reply rate ≥ 20% (теплый) / ≥ 5% (холодный) к 24 мая; (в) если ни один не набирает — пересмотр хука (не канала), поскольку канал может быть ОК, а проблема в сообщении.

### R-GS-2: Первый фотограф-евангелист не находится
**Вероятность:** средняя–высокая. **Impact:** высокий для Ставки 5. Весь photographer-led motion заточен под «первый чемпион = proof-point».
**Митигация:** (а) Маша фиксирует 3–5 имён до 10 мая (deadline в разделе 5); (б) fallback — **Маша-как-евангелист на собственных съёмках**: она сама коммерческий фотограф с клиентами, её собственный workflow становится первым кейсом. Это не идеально (меньше объективности), но это работает как directional proof; (в) если к концу июня нет внешнего евангелиста — перенос первого внешнего в Q3 с fallback Маша в Q2 retro.

### R-GS-3: Продуктовые хуки (автопереименование, share-link, brand-signature) задерживаются в PA
**Вероятность:** средняя. **Impact:** высокий. Outbound обещает функции — продукт должен их давать.
**Митигация:** (а) QA-smoke перед каждой волной — блокирует рассылку, если падает; (б) если хук не готов — волна переносится, не отправляется; (в) в outbound мы изначально пишем только про то, что **уже работает** (автопереименование, share-link), brand-signature — nice-to-have для Q2, не blocker.

### R-GS-4: Version A каскад не завершён, outbound запустить нельзя
**Вероятность:** низкая–средняя (в `.auto-memory/project_core_message.md` и `agent-team-v2.md:181` ещё старая формулировка; это зона CoS).
**Impact:** средний. GS не отправляет ни одной внешней волны, пока любой внутренний документ говорит противоречиво Version A.
**Митигация:** (а) блокер зафиксирован в `marketing/status.md`; (б) deadline каскада 3 мая — согласован с CoS; (в) если к 10 мая не закрыто — эскалация в CoS + Маша, волна 1 переносится.

### R-GS-5: Monetization-matrix не утверждён — нельзя делать pack-offer [anchor client]
**Вероятность:** средняя. **Impact:** низкий для Q2 ([test user A] case может быть без цен), средний для Q3.
**Митигация:** В Q2 outbound и [test user A] case пишем **без конкретных цен** (beta-landing-style); как только matrix утверждён — добавляем pack-offer в one-pager.

### R-GS-6: Tone-misalignment — outbound звучит «не как Маша»
**Вероятность:** средняя. **Impact:** средний. Маша подписывает лично, но не она пишет.
**Митигация:** (а) каждое сообщение волны 1 проходит personal approval Маши (deadline 20 мая); (б) 2–3 итерации на первые 5 сообщениях, чтобы GS откалибровался; (в) dogfood-demo (раздел 5, пункт 6) даёт GS первичный опыт и голос.
