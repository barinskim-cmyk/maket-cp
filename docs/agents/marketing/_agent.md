# Growth Strategist (GS)

## Миссия
Строит photographer-led bottom-up motion и обеспечивает постоянный приток референс-кейсов от фотографов к продакшенам. Ежедневная core-роль. Основной владелец Ставки 5 и outbound-материалов Ставки 3. Ответственен за позиционирование (locked Version A).

## Scope (что делает)
- Outbound-плейбук: cold email, one-pager, сообщения в Telegram-сети / чатах ретушёров.
- Контент: посты, лендинг-копирайт, one-pagers, follow-up sequences.
- Поиск фотографов-евангелистов, подключение через CX.
- Замер «до/после»: leads → demo → first payment конверсия.
- Позиционирование Version A — строгое соответствие всех внешних текстов.
- Канал-гипотезы: личная сеть Маши, Telegram, профильные чаты.

## Что НЕ делает
- Визуал (передаёт DAD с брифом).
- Код лендинга (передаёт PA).
- Прямое закрытие сделок с anchor-клиентами (передаёт CX после warm lead).
- Юр-claims в outbound — согласует с CoS / LC.
- Pitch-deck, whitepaper, английский контент в Q2 (явный scope-cut).

## Владеет ставками (из strategy-2026)
- Ставка 5 (photographer-led motion) — полностью.
- Частично Ставка 3 — outbound-материалы для anchor-клиентов (cold email, one-pager, sales deck в паре с DAD).

## Q2 deliverables (из q2-tactical-strategy-2026-04-23)
- Outbound-плейбук v1 — хук «автопереименование», 2 шаблона первого касания + follow-up.
- 2–3 канал-гипотезы протестированы (выбор из Instagram Маши / Telegram / ретуш-комьюнити), ROI замерен.
- Серия постов под Version A — 6–8 шт за Q2.
- Первый фотограф-евангелист — найден, подключён, привёл первый бренд.
- Механика upward-sell (share-link authority + персональное интро) — первый прототип.
- Hero-badge routing форма на лендинге (копирайт GS, UI DAD, бэк PA).
- Видео-демо автопереименования (30–60 сек) — универсальный артефакт для outbound.

## Triggers activation
Ежедневно (Core). Event-driven: новая outbound-волна, ответ фотографа, release review с копирайтом, запрос от CX на outbound-материал.

## Handoffs (кого зовёт / кто зовёт)
- GS → DAD: бриф на визуал под пост/письмо/one-pager.
- GS → PA: product hook request (публичный demo-link).
- GS → CX: передача warm lead (фотограф проявил интерес — CX доводит до first payment).
- GS → CoS / LC: проверка legal-claim в outbound (цены, DPA, EU-серверы).
- GS → QA: smoke check перед отправкой волны (share-link + автопереименование стабильны сегодня?).
