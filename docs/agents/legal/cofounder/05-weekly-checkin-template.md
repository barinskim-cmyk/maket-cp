# 05 — Weekly check-in template

Template для еженедельной заметки Маши по результатам check-in call'а с кандидатом во время engagement-test'а.

- **Формат.** Google Docs или локальный markdown. Один файл на кандидата, одна секция на неделю.
- **Аудитория.** Только Маша. Кандидату НЕ показывается.
- **Длительность call'а.** 30-45 минут.
- **Частота заполнения.** Сразу после call'а, пока в памяти.

Версия шаблона: v1.0, 2026-04-24.

---

## Шаблон — копировать раз в неделю

```
## Неделя {N} — {DATE}

Кандидат: {CANDIDATE_NAME}
Задача: [anchor client] pack design
Call duration: ___ мин
Формат: video / audio / async (подчеркнуть)

---

### 1. Что сделано за неделю (кандидат рассказывает)

- [ ] Ключевой deliverable этой недели: ___
- Что закрыто полностью:
  - ___
  - ___
- Что продвинулось, но не закрыто:
  - ___
- Ссылки на артефакты (docs, deck drafts, outreach):
  - ___

### 2. Что заблокировано

- Блокер 1: ___ (причина: ___ / кто должен разблокировать: ___ / ожидаемый срок: ___)
- Блокер 2: ___
- Вопросы, на которые кандидат ждёт ответа от Маши:
  - ___

### 3. Следующие шаги (на следующую неделю)

Кандидат делает:
- ___
- ___

Маша делает (чтобы разблокировать кандидата):
- ___

Совместно:
- ___

---

### 4. Observation — 5 критериев (шкала 1-5, только для Маши)

Шкала: 1 = явно проблема / 2 = ниже ожидания / 3 = ок / 4 = хорошо / 5 = выдающе.

| Критерий | Оценка (1-5) | 1 предложение с примером |
|---|---|---|
| Responsiveness — скорость отклика, без напоминаний | | |
| Initiative — предлагает ли сам, или только выполняет | | |
| Quality — самостоятельно ли доводит, на каком уровне | | |
| Emotional fit — демпфер или усилитель (ощущение после call'а) | | |
| Strategic sense — видит бизнес-картину или только тактику | | |

---

### 5. Insights (что нового узнал / куда двигается продукт)

- Что кандидат предложил нового / поставил под сомнение:
  - ___
- Что изменилось в моём (Маши) понимании продукта/клиента из-за этой недели:
  - ___
- Внешние сигналы (от клиента, рынка, сети), которые всплыли:
  - ___

---

### 6. Decision / action items

- [ ] Action для Маши: ___
- [ ] Action для кандидата: ___
- [ ] Решение, которое нужно принять до следующей недели: ___
- [ ] Escalation (если блокер высокого приоритета): ___

---

### 7. Финальное ощущение (одно предложение)

После этого call'а я чувствую: ___

Это сигнал — про что?
___
```

---

## Инструкция Маше по заполнению

**Что важно делать:**

- Заполнять **сразу после call'а**, не откладывать. В памяти — детали, через день — только общее впечатление.
- В Observation (секция 4) **оценивать не поведение в моменте**, а паттерн за неделю. Один плохой день ≠ 2/5 по responsiveness.
- В «1 предложение с примером» — **факт, а не оценка.** «Ответил через 4 часа на сообщение от 14:00 во вторник» лучше, чем «был отзывчивый».
- Читать observations **вертикально** (всю колонку по неделям). Паттерн виден в динамике, не в одной строке.
- Секция 7 («Финальное ощущение») — самая важная в долгосрочной перспективе. Это то самое «демпфер или усилитель» — фиксируется честно, без приукрашивания.

**Что НЕ делать:**

- НЕ показывать эту заметку кандидату. Эффект наблюдателя обнулит наблюдения.
- НЕ пропускать сложные недели — особенно ценно, когда есть, о чём писать плохое.
- НЕ сводить все оценки к 3/5 («вроде нормально»). Либо 2, либо 4 — Маша должна давать себе право на мнение.
- НЕ переписывать прошлые недели задним числом. Если ощущение изменилось — фиксировать как insight, не как редактирование истории.

## Сводная таблица по результатам теста

В конце 4-5 недели — собрать все weekly-заметки и заполнить единую таблицу наблюдений:

```
Кандидат: {CANDIDATE_NAME}
Задача: [anchor client] pack design
Общая длительность: {N} недель

| Неделя | Responsiveness | Initiative | Quality | Emotional fit | Strategic sense | Заметка |
|---|---|---|---|---|---|---|
| 1      |                |            |         |               |                 |         |
| 2      |                |            |         |               |                 |         |
| 3      |                |            |         |               |                 |         |
| 4      |                |            |         |               |                 |         |
| 5      |                |            |         |               |                 |         |

### Паттерн за весь тест
- Responsiveness: динамика (улучшается / стабильна / ухудшается)
- Initiative: ___
- Quality: ___
- Emotional fit: ___
- Strategic sense: ___

### Must-have pass/fail

- [ ] Responsiveness: pass / fail (must be pass)
- [ ] Quality of output: pass / fail (must be pass)
- [ ] Emotional fit as demper: pass / fail (must be pass)

Если хоть одно fail — **не берём** (см. cofounder-package.md, раздел 7).

### Nice-to-have

- Strategic sense: ___
- Proactivity: ___
- Доп. активы (US-bridge, networking и т.д.): ___

### Red flags наблюдались?

- [ ] Verbal yes без действий — ___
- [ ] Эмоциональное усиление вместо демпфирования — ___
- [ ] Требование формализации с первой недели — ___
- [ ] Попытка доминировать в продуктовых решениях — ___
- [ ] Личный багаж в работу — ___

### Решение

Рекомендация для decision conversation: **брать / не брать / альтернативный формат** (fractional / advisor / сотрудничество вне cofounder-формата).

Обоснование в 3 предложениях: ___
```

Эта сводная таблица — input для decision conversation (см. cofounder-package.md, раздел 9.3).

---

## Placeholders

- `{N}` — номер недели (1, 2, 3...).
- `{DATE}` — дата call'а в формате YYYY-MM-DD.
- `{CANDIDATE_NAME}` — имя кандидата.
