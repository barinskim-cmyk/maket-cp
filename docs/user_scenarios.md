## ФОТОГРАФ (Desktop)

### 1. Создание проекта и загрузка фотографий -- все готово (7/7)
Открыть приложение, создать съемку, выбрать шаблон, перетащить папку с фото, авто-масштабирование, авто-определение ориентации, автосохранение каждые 2 сек.

### 2. Работа с превью-панелью -- 6/7 готово
Готово: ресайз панели, слайдер колонок, фильтры по папке и рейтингу, полноэкранный просмотр, ZIP экспорт.
Не готово: горизонтальные фото на 2 колонки в сетке.

### 3. Наполнение карточек товаров -- 6/8 готово
Готово: добавление карточек, drag-and-drop фото, переключение ориентации, hero-слот, поворот фото, удаление.
Не готово: несколько вариантов фото в один слот (options[]), карусель вариантов.

### 4. Управление шаблонами карточек -- 3/5 готово
Готово: создание шаблона из карточки, редактирование, сохранение в библиотеку.
Не готово: библиотека шаблонов (CRUD), применение шаблона ко всем карточкам.

### 5. Дополнительный контент -- 3/5 готово
Готово: drag-and-drop в секцию, 1200px качество, удаление.
Не готово: теги каналов (WB, Ozon...), фильтрация по каналу.

### 6. Пайплайн (этапы съемки) -- 3/5 готово
Готово: визуальная шкала 8 этапов, подсветка активного, таймстемпы.
Не готово: триггеры автоперехода, подсказки по этапам.

### 7. Rate Setter -- 5/8 готово
Готово: режим текст, режим папка, выбор сессии C1, dry-run, лог выполнения.
Не готово: режим "из карточек" (автосбор имен), запись ключевых слов в COS, цветные метки.

### 8. Облачная синхронизация -- все готово (9/9)
Готово: настройка Supabase, авторизация (email + magic link), регистрация, автосинхронизация (debounce 3 сек), загрузка превью, скачивание проекта, список облачных проектов.
Новое (v2.2): непрерывная авто-синхронизация desktop (как web), оффлайн-режим (localStorage + авто-sync при reconnect).

### 9. Публикация клиентской ссылки -- все готово (4/4)
Готово: создание share-ссылки по ролям, копирование, список ссылок, работает через GitHub Pages.

### 10. Экспорт -- 2/4 готово
Готово: CSV, PDF (jsPDF).
Не готово: ТЗ на ретушь (фото + аннотации + комментарии), адаптация под каналы.

### 11. Артикулы и переименование -- 3/4 блоков готово (новое v2.2)
Готово: загрузка чек-листа (JSON/ручной ввод), сопоставление артикулов с карточками, верификация.
Не готово: блок 4 -- переименование через Rate Setter (запись нового имени в COS).
Также: импорт legacy-проектов, статистика прогресса.
