# Maket CP -- Мобильный режим клиента

**Концепт-документ | v1.2 | 01.04.2026**

---

## Проблема

Клиент получает ссылку для просмотра и редактирования отбора фотографий в карточках товара. Текущий интерфейс рассчитан на десктоп: сетки карточек, drag-and-drop слотов, панели превью не адаптированы под мобильные экраны. Клиенты часто открывают ссылки на телефоне. Им нужны два основных действия: просмотреть все фотографии и заменить/убрать фото в карточках, а в конце -- согласовать или вернуть на доработку.

---

## Общая структура

Два режима, переключаемые одной кнопкой в фиксированной шапке:

- **Шапка (sticky):** название проекта + кнопка "SHOW GALLERY" / "GO BACK TO CARDS"
- **Режим карточек:** непрерывная вертикальная лента из всех карточек
- **Режим галереи:** сетка всех превью с галочками и увеличением

---

## Режим карточек (основной)

### Раскладка ленты

Непрерывный скролл вниз. Все карточки идут подряд, между ними -- разделитель с номером ("Карточка 3 / 36"). Внутри каждой карточки фото располагаются по правилам:

| Тип фото | Раскладка |
|---|---|
| **Заглавное (hero, weight >= 2)** | На всю ширину экрана, одно в ряду |
| **Незаглавная вертикаль** | По 2 в ряду. Если в конце нечётное количество -- последняя занимает 50% слева, рядом пустой слот (можно добавить фото). |
| **Горизонталь** | На всю ширину экрана |

### Замена фото (карусель в слоте)

Свайп по фото влево/вправо (или стрелки) -- листает ближайшие ~10-20 фотографий из галереи проекта относительно текущего.

| Параметр | Описание |
|---|---|
| **Фильтр по ориентации** | Горизонтальный слот листает только горизонтальные фото. Вертикальный -- только вертикальные. |
| **Источник фото** | Ближайшие фото из галереи относительно текущего (по имени файла = порядок съёмки) |
| **Диапазон** | ~10-20 фото в каждую сторону от текущего |
| **Дубликаты** | Разрешены. Клиент может поставить одно фото в несколько слотов. |

### Быстрое заполнение пустого слота

Двойной тап на пустой слот -- автоматически вставляет следующее фото из галереи (по порядку после первой вертикали в этой карточке, с учётом ориентации слота). Быстрый способ заполнить слот без листания карусели.

### Удаление фото

Кнопка X на каждом слоте. Слот становится пустым. Не часть карусели.

### Завершение просмотра

В конце ленты (после последней карточки) -- две кнопки:

- **"Согласовать"** -- клиент одобряет отбор, проект переходит на следующий этап пайплайна
- **"Вернуть на доработку"** -- клиент возвращает проект фотографу с комментариями

Это замыкает цикл пайплайна: фотограф отправляет -> клиент смотрит -> согласует / возвращает.

### Авто-синхронизация

Каждое изменение (замена фото, удаление) запускает отложенную синхронизацию в Supabase (задержка 3 сек, мьютекс). Финальное решение ("Согласовать" / "Вернуть") меняет stage проекта.

## Режим галереи

| Параметр | Описание |
|---|---|
| **Раскладка** | Адаптивная сетка с flex-grow (без пустот). 3 колонки для вертикалей, горизонталь заполняет ряд. |
| **Полный экран** | Нажатие -- увеличение. Свайп влево/вправо в полноэкранном режиме. |
| **Галочки** | Клиент может отметить фото для "доп. контента" (дополнительная ретушь). Галочка видна на миниатюре. |
| **Фильтры** | Фильтр по рейтингу (звёзды) и по папке -- как на десктопе. |
| **Авто-синхронизация** | Отметки синхронизируются в Supabase (колонка projects.other_content). |

## Техническая реализация

### Подход

Чисто фронтенд. Никаких изменений бэкенда или схемы Supabase (кроме обновления stage при согласовании). Адаптивный CSS для viewport < 768px. Десктопный опыт не меняется -- мобильный режим активируется через media query.

### Ключевые функции

- `cpMobileRenderFeed()` -- рендер непрерывной ленты карточек
- `cpSlotCarousel(slotIdx, direction)` -- переход к следующему/предыдущему фото в слоте
- `cpGetNearbyPreviews(currentFile, orient, range)` -- возвращает ~20 ближайших фото по ориентации
- `cpClientApprove()` / `cpClientReject()` -- согласовать / вернуть на доработку (меняет stage)
- `pvMobileRenderGallery()` -- мобильная галерея с галочками
- Touch-свайп через Pointer Events API

### Затронутые файлы

- `v2/frontend/css/style.css` -- адаптивные стили, мобильная лента
- `v2/frontend/js/cards.js` -- мобильный рендер, карусель, согласование
- `v2/frontend/js/previews.js` -- мобильная галерея, touch-события
- `v2/frontend/js/supabase.js` -- обновление stage при согласовании/возврате
- `v2/frontend/index.html` -- viewport meta tag

## Вне скоупа (мобильный)

- Редактор шаблонов -- только десктоп
- Drag-and-drop между слотами -- заменён каруселью
- Загрузка превью -- только фотограф (десктоп)
- Rate Setter -- только фотограф (десктоп)

## Открытые вопросы

| Вопрос | Текущий ответ |
|---|---|
| **Создание/удаление карточек на мобильном?** | Пока вне скоупа. Если понадобится -- добавим кнопку "+" и свайп для удаления. |
| **Комментарии при возврате?** | При нажатии "Вернуть на доработку" -- показать поле для текстового комментария. |
| **Точка перелома (breakpoint)?** | 768px. Ниже = мобильная лента. Выше = десктоп. |
| **Работа оффлайн?** | Поддерживается. Изменения сохраняются в localStorage, авто-синхронизация при reconnect (событие `online`). Реализовано в shootings.js (shAutoCloudSync, shSyncPendingProjects). |
