# Maket CP -- Инструмент артикулов (Rename Tool)

**Проектный документ | v1.0 | 01.04.2026**

---

## Назначение

Инструмент для сопоставления артикулов из чек-листа клиента с карточками товара в проекте. Решает проблему ручного переименования фотографий: фотограф снимает в своей нумерации (IMG_0001), клиенту нужны файлы с артикулами (ECO-1234-BLK). Инструмент связывает артикул с карточкой, проверяет полноту, а затем Rate Setter переименовывает файлы в Capture One.

---

## Архитектура

Фронтенд-модуль `articles.js` с префиксом `ar*`. Четыре блока, последовательный workflow.

### Блок 1 -- Загрузка чек-листа (реализован)

Загрузка списка артикулов. Источники:

- JSON-файл (drag-and-drop или file picker)
- Excel/CSV (парсинг через PapaParse / SheetJS -- план)
- Ручной ввод (форма для быстрого добавления)

Каждый артикул (Article entity):
```
{
  id: number,         // порядковый номер
  sku: string,        // артикул (ECO-1234-BLK)
  category: string,   // категория (обувь, сумки, ...)
  color: string,      // цвет
  refImage: string,   // URL референса (опционально)
  status: string,     // unmatched | matched | verified
  cardIdx: number     // индекс связанной карточки (-1 = не сопоставлен)
}
```

UI: таблица артикулов со статусами, кнопка импорта, статистика (сколько сопоставлено / всего).

### Блок 2 -- Сопоставление (реализован)

Связывание артикулов с карточками проекта. Две колонки:

- Слева: список несопоставленных артикулов (кликабельные)
- Справа: список карточек проекта (кликабельные)

Workflow: выбрать артикул -> выбрать карточку -> нажать "Сопоставить". Статус артикула меняется на `matched`, привязывается `cardIdx`.

Будущее: AI Vision (OpenAI API) -- автоматическое сопоставление по референсному изображению из чек-листа и фотографиям в карточке.

### Блок 3 -- Верификация (реализован)

Проверка корректности сопоставления. Таблица:

| Артикул | Карточка | Фото | Статус |
|---------|---------|------|--------|
| ECO-1234-BLK | Карточка 5 | hero.jpg, img2.jpg | [Подтвердить] |

Workflow: фотограф просматривает каждую пару артикул-карточка, подтверждает (verified) или отменяет (unmatched). Кнопка "Подтвердить все" для быстрого подтверждения.

### Блок 4 -- Переименование через Rate Setter (план)

Финальный шаг. После верификации Rate Setter получает список:
```
stem: IMG_0001 -> new_name: ECO-1234-BLK_01
stem: IMG_0002 -> new_name: ECO-1234-BLK_02
```

Rate Setter записывает новое имя в COS-файл Capture One. Требует Python-бэкенд (только desktop mode).

## AI Vision (план)

Интеграция с OpenAI Vision API для автоматического сопоставления:

1. Клиент загружает чек-лист с референсными изображениями товаров
2. AI анализирует фото в карточках и сравнивает с референсами
3. AI предлагает сопоставление (автоматически заполняет cardIdx)
4. Фотограф проверяет и подтверждает в блоке 3 (верификация)

Требует: API-ключ OpenAI, хранение референсных изображений в Supabase Storage.

## Особые случаи

- **Shared photos** (фото в нескольких карточках): при переименовании получают двойное имя (ECO-1234_ECO-5678)
- **Лишние карточки** (без артикула): пропускаются, помечаются предупреждением
- **Лишние артикулы** (без карточки): остаются в статусе unmatched, видны в статистике
