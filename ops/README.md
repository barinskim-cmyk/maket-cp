# /ops/

Операционные playbook'и для CoS и команды. Живые рабочие документы, не дубликат стратегий.

## Содержимое

- `automation-rules.md` — T-01..T-07 с триггерами, действиями, failure recovery и примерами.
- `release-review-playbook.md` — матрица состава review под тип релиза, status card template, escalation.
- `runbook-[anchor client]-downtime.md` — операционный runbook при инциденте на [anchor client] (совместно с PA, добавится в неделе 10).
- `strategy-update-YYYY-MM-DD.md` — cascade-запросы на все лейны при смене стратегии (появляется по триггеру).

## Правила

1. Playbook — это то, к чему CoS обращается в момент триггера. Если его нет под рукой за 1 минуту — он не работает.
2. Обновление по retro: в конце Q2 CoS пересматривает автоматизации и фиксирует, какие сработали, какие стали лишними.
3. Автоматизации — не бюрократия. Если правило не ловит реальные случаи за 2 цикла подряд — удалить или переформулировать.
4. Для каждого правила — пример сценария. Пример — критерий проверки формулировки.

## Связанные артефакты

- `agent-team-v2.md` разделы "Automation rules" и "Execution loop" — источник правды.
- `/project-log.md` — триггеры и их срабатывания логируются сюда.
- `/docs/agents/ops/` — стратегические артефакты CoS (q2-strategy, audit, status).
