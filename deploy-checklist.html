Почему Netlify, а не хостинг REG.RU?
Наш фронтенд -- это статические файлы (HTML, JS, CSS). Для них не нужен PHP или сервер.
Netlify (рекомендую)
+ Бесплатно навсегда (100 ГБ трафика/мес)
+ Деплой за 2 минуты -- просто перетащить папку
+ Автоматический HTTPS (SSL сертификат)
+ CDN по всему миру -- сайт грузится быстро
- Нужно зарегистрироваться на netlify.com
Хостинг REG.RU
+ Уже есть аккаунт
- Платный (от ~150 руб/мес)
- Сложнее настроить (FTP, панель управления)
- SSL надо настраивать отдельно
Рекомендую Netlify. Ниже инструкция для него. Если хочешь использовать хостинг REG.RU -- напиши, сделаю инструкцию для него тоже.
1. Netlify -- регистрация и деплой
Весь процесс занимает ~5 минут
Открой app.netlify.com -- можно войти через GitHub, Google или email.
1. На главной странице Netlify найди блок "Want to deploy a new site without connecting to Git? Drag and drop your site output folder here"
2. Открой папку проекта: Maket CP / v2 / frontend
3. Перетащи ВСЮ папку "frontend" в поле на сайте Netlify
4. Подожди 10-30 секунд -- сайт опубликуется
Netlify даст временный адрес вида random-name-12345.netlify.app
Открой его -- должен появиться Maket CP.
Site configuration > Site details > Change site name
Например: maketcp -- тогда адрес будет maketcp.netlify.app
2. Подключить домен с REG.RU
Направляем твой домен на Netlify через DNS
Какой домен будем использовать? Можно основной (maketcp.ru) или поддомен (app.maketcp.ru)
1. В Netlify: Site configuration > Domain management > Add custom domain
2. Введи свой домен (например maketcp.ru)
3. Netlify скажет "Check DNS configuration" -- это нормально, настроим дальше
1. Зайди в REG.RU > Мои домены > выбери домен > DNS-серверы и управление зоной
2. Добавь записи:

Для ОСНОВНОГО домена (maketcp.ru):
-- Удали старые A-записи для @ (если есть)
-- Добавь A-запись: @ -> 75.2.60.5
-- Добавь CNAME-запись: www -> [твой-сайт].netlify.app

Для ПОДДОМЕНА (app.maketcp.ru):
-- Добавь CNAME-запись: app -> [твой-сайт].netlify.app
75.2.60.5 -- это IP-адрес Netlify для A-записей основного домена. DNS обновляется от 5 минут до 48 часов (обычно 15-30 минут).
Netlify автоматически выпустит SSL-сертификат после того как DNS обновится.
Site configuration > Domain management > HTTPS -- должно быть "Your site has HTTPS enabled"
Если не обновилось -- подожди 30 минут и нажми "Verify DNS configuration"
Открой https://твой-домен -- должен появиться Maket CP.
Проверь что HTTPS работает (замочек в адресной строке).
3. Обновить Supabase -- разрешить новый домен
Чтобы авторизация работала с нового домена
1. Supabase Dashboard > Authentication > URL Configuration
2. В "Redirect URLs" добавить:
-- https://твой-домен.ru
-- https://твой-домен.ru/**
-- https://www.твой-домен.ru (если используешь www)
3. В "Site URL" поставить: https://твой-домен.ru
Открой сайт по домену, попробуй войти через email+пароль или magic link.
Должно работать без ошибок.
4. Как обновлять сайт потом
Когда мы вносим изменения в код
Каждый раз когда мы обновляем код:
1. В Netlify: Deploys > внизу "Need to update your site? Drag and drop your site output folder here"
2. Перетащи папку frontend заново
3. Через 10 секунд сайт обновится

Потом можно подключить Git (GitHub) для автодеплоя -- тогда обновления будут автоматические.
Когда сайт заработает по домену -- скинь мне URL, я проверю что всё ок и обновлю настройки в коде.
