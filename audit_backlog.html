Контент — Карточки товара
Сайдбар карточек Готово 2. Отбор фотографа —
  • Список карточек (номер, категория, кол-во фото)
  • Добавление новой карточки
  • Экспорт списка (.txt)
  • Выделение активной карточки
  • Быстрый доступ к любой карточке
  • Видно заполненность каждой карточки
  • cards.js: cpRenderList(), cpAddCard()
  • Слоты: null = пустой, {file, dataUrl} = заполнен
 Нет drag-and-drop для сортировки карточек. Нет группировки/поиска. Экспорт только в .txt (нет PDF-экспорта карточек из v2).
Редактор карточки (EKONIKA-стиль) Готово 2. Отбор фотографа Все карточки заполнены → можно перейти к «Отбор клиента»
  • Раскладка: 1 горизонтальная + 2–3 вертикальных
  • Drag-and-drop: из ОС, из превью, между слотами (swap)
  • Удаление фото из слота
  • Переключатель 2/3 вертикали
  • Выбор категории товара
  • Undo (до 10 шагов, Cmd+Z)
  • Навигация стрелками между карточками
  • Карточка — центральная сущность: фотограф видит как кадры работают вместе
  • Drag-and-drop из превью — основной способ работы
  • Undo — страховка от ошибок
  • cards.js: cpRenderCard(), cpSlotHTML(), cpBindSlotEvents()
  • Drop priority: 1) application/x-preview, 2) OS files, 3) slot swap
  • History: JSON-снэпшоты slots
  • Desktop: хранит path, thumb для display; full image через get_full_image
 Нет опций (вариантов) в слоте — по документации запланировано (карусель, выбор клиента). Нет комментариев к карточке. Шаблон фиксирован (горизонт + вертикали), нет кастомных шаблонов. Нет PDF-экспорта из v2.
Панель превью (справа) Готово 1. Преотбор 2. Отбор —
  • Загрузка: drag-and-drop папки/файлов + кнопка «Выбрать папку»
  • Прогресс загрузки (X / Y, %)
  • Миниатюры: canvas (браузер) или Pillow (desktop)
  • Фильтр по рейтингу (XMP из Capture One)
  • Фильтр по папке (при загрузке нескольких папок)
  • Дедупликация по имени файла
  • Зелёная галочка на фото, уже используемых в карточках
  • Drag из превью в слоты карточек
  • Lazy rendering (60 шт + скролл)
  • Сортировка по имени (слева-направо, сверху-вниз)
  • Быстрый просмотр всех превью
  • Рейтинг из Capture One сохраняется
  • Видно что уже отобрано, что нет
  • Несколько папок без дубликатов
  • previews.js: pvRenderPanel(), pvBuildHTML(), pvReadXmpRating()
  • preview_service.py: scan_folder(), _read_xmp_rating()
  • XMP: sidecar .xmp, embedded (256KB), Pillow getxmp()
  • Модель: {name, path, thumb, rating, folders}
  • Concurrent loading: 4 потока
 Рейтинг read-only (не пишется обратно в XMP). Нет полноэкранного просмотра фото. Нет color label из Capture One. При очистке превью не спрашивает про карточки.

  • Два режима: список имён (textarea) или папка-источник
  • Выбор папки сессии Capture One
  • Удаление суффиксов (_preview, _web...)
  • Пробный запуск (dry-run)
  • Лог в реальном времени (push из Python)
  • Цветовая разметка: ok/miss/err/skip/dry
  • Результат: обновлено / не найдено
  • Автоматическое выставление рейтинга 5 звёзд на утверждённые фото в Capture One
  • Экономит часы ручного поиска
  • sync.js: runRateSetter(), push-лог
  • rate_setter.py: run(), clean_name(), collect_source_stems()
  • cos_repository.py: build_index(), update_rating()
  • Атомарная запись, .cos.bak бэкапы
 Не привязан к карточкам: можно было бы автоматически собирать список имён из заполненных карточек. Нет кнопки «Синхронизировать из карточек».

Persistence (save/load) Частично Все —
  • Save/load проекта через файловый диалог (JSON)

  • Не терять работу
  • Возобновлять сессию

 Нет автосохранения. Нет сохранения превью-пула (только карточки). base64 в JSON — тяжёлые файлы. Нет браузерного persistence (IndexedDB убрали).
